# 📊 AdventureWorks Sales Performance Dashboard

An end-to-end Power BI dashboard designed to analyze sales performance, track key financial metrics (KPIs), and uncover actionable business insights for the AdventureWorks dataset.

---


## 🎯 Business Objectives
The primary goal of this dashboard is to help stakeholders and business owners answer critical questions:
* **Financial Health:** What is our Total Due, and how do Sub-totals, Tax, and Freight impact our margins?
* **Sales Trends:** How is our order volume evolving year-over-year? Where are the peak seasons?
* **Product Performance:** Which product categories drive the most volume and tax contributions?
* **Geographical Distribution:** Which territories have the highest customer concentration?

---

## 🛠️ Tech Stack & Skills Demonstrated
* **Tool:** Power BI Desktop
* **Data Source:** AdventureWorks Dataset
* **Data Transformation:** Power Query (Data cleaning, handling nulls, changing data types).
* **Data Modeling:** Star Schema / Snowflake Schema implementation with clear Fact and Dimension relationships.
* **Analytical Calculations:** DAX (Data Analysis Expressions) for dynamic KPIs and time-intelligence metrics.
* **Data Visualization:** Creating a clean, user-friendly UI/UX using corporate-style layout and consistent color palettes.

---

## 🔍 Key Insights Uncovered
1. **The 2013 Spike:** Order volume peaked dramatically in 2013 (reaching over 708K orders) before experiencing a sharp decline in 2014. This indicates a need to investigate seasonal promotions or market anomalies in 2013.
2. **Product Dominance:** The **Bikes** category is the undisputed primary revenue and tax driver, while **Accessories** and **Clothing** represent minor shares.
3. **Top Territories:** **Canada** and the **Northwest** regions hold the largest customer base, making them prime targets for localized marketing campaigns.

---

## 📂 How to Run the Project
1. Clone this repository to your local machine.
2. Ensure you have **Power BI Desktop** installed.
3. Open the `Sales_Performance_Dashboard.pbix` file.
4. *(Optional)* If data links are broken, re-route the data source settings to the files inside the `Data/` folder.
